# 投资人思维框架提炼方法论

> 从原始信息到可运行的投资人心智模型，核心方法论

## 一、调研 Input 维度（投资人特化）

在提炼投资人心智模型前，必须覆盖以下核心信息源：

1. **过往投资案例（核心）**：
   - 投了哪些项目？（成功案例与失败案例）
   - 投资时的阶段、估值、核心逻辑是什么？
   - 退出时的回报率如何？
2. **加入投资机构后的公开表态**：
   - 在机构官网、年度信、LP报告中的发言。
   - 在行业大会、播客访谈中的观点。
3. **社交媒体发言（加入机构后）**：
   - X/Twitter、LinkedIn 上的高频互动、点赞、转发和原创内容。
   - 重点关注其对特定赛道、创始人特质、宏观经济的即时反应。

## 二、心智模型识别的三重验证（投资决策特化）

一个论点要被认定为「投资人心智模型」而非「随口一说」，必须通过三重验证：

### 验证1: 跨案复现
同一个思维框架出现在此人主导的至少2个不同投资案例中。

例：如果「重仓创始人」是心智模型——
- 在投A项目时，因为创始人特质忽略了早期数据的不足。
- 在投B项目时，因为创始人特质在竞争激烈的赛道中下注。
→ 跨2个案例复现 → 这是真正的心智模型

### 验证2: 有生成力
用这个模型可以推断此人对新赛道或新项目的可能立场。

例：如果「网络效应优先」是心智模型——
- 面对「SaaS工具」→ 他会先看是否有数据飞轮。
- 面对「AI应用」→ 他会先看是否有用户生成内容的网络效应。
→ 能生成新推断 → 这是真正的心智模型

### 验证3: 有排他性
不是所有投资人都会这样想，这个模型体现了此人的独特视角（Alpha的来源）。

例：「反共识且正确」是霍华德·马克斯的，不是所有人都这样看市场。
→ 有区分度 → 这是值得提炼的心智模型

**如果一个观点只通过了1重验证** → 降级为「决策启发式」而非「心智模型」
**如果一个观点0重验证都没过** → 可能只是此人在特定场景的公关话术，不纳入

## 三、表达DNA的量化方法

### 3.1 句式指纹

从此人的长文/演讲/推文中随机抽取20个段落，统计：

| 维度 | 测量方式 |
|------|---------|
| 平均句长 | 字数/句数 |
| 疑问句比例 | 疑问句数/总句数 |
| 类比密度 | 类比数/千字（投资人常喜欢用体育/战争类比） |
| 第一人称使用率 | 「我」出现频率 |
| 确定性语气比例 | 「一定」「显然」vs「也许」「可能」 |
| 财务术语密度 | ARR、LTV/CAC、PMF等词汇频率 |

### 3.2 风格标签

从以下维度打标：

```
正式 ←→ 口语
抽象（看宏观） ←→ 具体（看微观数据）
谨慎（看风险） ←→ 断言（看收益）
学术（看理论） ←→ 通俗（看常识）
长句 ←→ 短句
铺垫型 ←→ 结论先行
数据驱动 ←→ 叙事驱动（看重故事）
```

### 3.3 禁忌词和口癖

- 此人从不用的词 → 生成Skill时也不用（如有些投资人从不说"颠覆"）
- 此人的口癖/高频表达 → 适度使用（太多变成模仿秀）

## 四、矛盾处理原则

投资人的矛盾往往是其投资哲学的核心张力。

### 三种矛盾类型

1. **时间性矛盾**（观点演化）
   - 此人早期看好消费，后来只看好企业服务。
   - 处理：记录演化轨迹，标注「早期」「近期」。在Skill中以「近期观点」为主，但提及演化。

2. **领域性矛盾**（不同赛道不同规则）
   - 此人在投SaaS时看重稳健增长，在投Crypto时看重爆发力。
   - 处理：分赛道记录，不强求统一。

3. **本质性张力**（投资哲学的内在冲突）
   - 例：既追求高胜率又追求高赔率。
   - 处理：明确记录为「核心张力」。这通常是此人最有意思的部分。

## 五、质量自检清单

生成Skill后，用以下问题自检：

### 心智模型
- [ ] 每个模型都有至少2个不同投资案例的证据？
- [ ] 模型数量在3-7个之间？
- [ ] 每个模型都有明确的适用赛道和局限？

### 决策启发式
- [ ] 聚焦于看项目、看创始人、看赛道的具体逻辑？
- [ ] 每条规则都有具体投资案例支撑？

### 表达DNA
- [ ] 读起来像个真实的投资人，而不是通用AI？
- [ ] 抓住了其独特的财务/商业词汇偏好？

### 诚实边界
- [ ] 明确写了不投什么赛道？
- [ ] 标注了信息源和调研时间？
